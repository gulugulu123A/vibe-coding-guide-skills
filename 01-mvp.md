# 里程碑 1：先把页面跑起来

## 要做什么

让 AI 生成一个最简单的页面：能渲染、能滚动、能部署。别管好看不好看，先让它活着。

## 对话模板

> 帮我搭一个最简单的个人网站，要求：
> - 只有一个页面，从上到下包含：名字/头像区域、关于我、技能、项目经历、联系方式
> - 内容先用占位文字，以后我会替换
> - 用 [你的框架，如 Next.js / 纯 HTML / Vue] 来做
> - 做完后告诉我怎么在本地运行和部署到线上
>
> 保持最简单，不要任何动画和特效。

## 确认清单

- [ ] 本地能跑起来，浏览器打开能看到完整页面
- [ ] 部署到线上，链接能打开（手机也能访问）
- [ ] 页面可以正常滚动，没有布局错乱
- [ ] 我没有纠结任何关于颜色、字体、动效的问题——那些是后面的事

## 一个重要的心态

如果 AI 生成的代码你一行都看不懂，**没关系**。现在的目标是"让它跑起来"，不是"弄懂每一行"。

但如果 AI 三次都没修好同一个 bug，把报错信息直接复制给 AI，加一句：

> 这个报错已经出现三次了，帮我分析根因，不要绕过去。

## 停下来想想

**你在练什么：MVP 思维。**

MVP 是 Vibe Coding 最重要的节奏感——**先跑通，再迭代**。很多人一上来就想做完美的设计、炫酷的动效、精心打磨的内容，结果三天过去了还没部署过一次。

这对应六个特质里的**忍受不确定性**：不求完美，先求存在。页面是丑的、内容是假的、什么效果都没有——但它活着。从"不存在"到"存在"，是最难的一步。你已经跨过来了。
