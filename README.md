# Vibe Coding 新手上路指南 skills

> 从零搭一个个人网站，学会和 AI 高效协作。

## 什么是 Vibe Coding？给纯新手的解释

如果你是第一次听到这个词，下面几个问题应该能帮到你。

### 1. 这个词到底是什么意思？

"Vibe" 是氛围、感觉，"Coding" 是编程。合在一起的大白话就是：**你不用写代码，你用说话的方式告诉 AI 你想要什么，AI 帮你写。**

就像你给设计师描述你想要的装修风格一样——你不需要会画图纸，但你得说清楚"我要暖色调、木地板、大落地窗"。Vibe Coding 里，AI 就是那个帮你画图纸的人。

### 2. 我需要会写代码吗？

不需要。但你得愿意做三件事：**说清楚你要什么、看得出好坏、愿意反复改。**

这三件事听起来简单，其实每一样都需要练。很多人卡在第一步——脑子里有一个模糊的画面，但说不出来。这本指南的很大一部分就是在帮你练这个能力。

### 3. 我能用 Vibe Coding 做出什么东西？

网页、小程序、自动化脚本、个人博客、后台管理系统…… 基本上你日常能看到的软件产品，都能用 Vibe Coding 做出一个版本。不一定完美，但一定**能用**。

这本指南选择"搭一个个人网站"作为你的第一个项目，因为它最适合入门——你一边做一边就能看到效果，而且做完马上就能用。

### 4. 它跟"用 ChatGPT 问问题"有什么区别？

用 ChatGPT 问问题是"一次性"的——你问，它答，结束。

Vibe Coding 是"持续协作"——你打开一个 AI 编程工具，跟 AI 来回对话好几个小时甚至好几天，一起把一个项目从零搭建出来。你不会只问一个问题就走，你会反复追问、调整、修改、调试，像跟一个搭档合作一样。

### 5. 为什么需要学"方法"？不是直接聊就行了吗？

直接聊当然可以。但你大概率会撞上同一个问题：**一开始很顺，做到一半卡死了。**

原因通常不是 AI 变笨了，是你一次塞给它的东西太多了。AI 的上下文是有限的，就像一个只能记住最近 20 分钟对话的人。你如果上来就让它做一整个网站，它一定会漏掉东西。

这就是为什么**方法很重要**——知道怎么拆任务、怎么一步步推进、怎么在出错时不慌，这些才是 Vibe Coding 真正的门槛。

### 6. 我需要什么工具？要花钱吗？

目前主流的 AI 编程 Agent 有这些：

| 工具 | 特点 |
|---|---|
| **Claude Code** | 目前综合体验最好的编程 Agent，可用 VS Code 插件 |
| **Cursor** | 图形化界面，适合不习惯命令行的用户 |
| **GitHub Copilot** | 深度集成 VS Code，适合已有开发经验的用户 |
| **Codex (OpenAI)** | OpenAI 出品，云端运行 |

**新手入门推荐方案：Claude Code + 国产大模型**

Claude Code 的官方模型在国内访问不太方便，但它支持接入第三方模型。你可以搭配 DeepSeek、GLM、MiniMax 等国产大模型使用，价格便宜，中文理解也更好，入门成本很低。

具体安装方式各家不同，推荐卡兹克这篇教程，新手跟着走一遍就能装好：  
https://mp.weixin.qq.com/s/AA2NHww4jUBuAfi10EYICw

不需要买任何付费教程，网上免费的优质内容已经足够了。

### 7. 这本指南怎么帮我？

这本指南不是教你写代码的，是教你**跟 AI 协作的方法**。

它用"搭一个个人网站"作为你的第一个练手项目，拆成 7 个步骤，每一步都告诉你：这一步要做什么、怎么跟 AI 说、做完检查什么、以及这一步在练什么能力。

跟着走一遍，你带走的不只是一个网站，是一套以后做任何项目都能套用的方法。

### 8. 作者寄语

最后想说的是，不用觉得 Vibe Coding 有多高的门槛。它和学习任何新东西一样，最好的方式就是先迈一步试试。

市面上的教程很多，内容扎实但篇幅也长。我们建议不需要全部啃完再动手——花一点时间了解基础概念，然后直接上手做个小项目，遇到问题再回头查。带着问题学，往往比死记硬背来得快。

做中学，学中做。这是我们觉得最适合入门 Vibe Coding 的节奏。当然，如果你习惯先系统学完再动手，也完全没问题，按自己的节奏来就好。

希望这本指南能帮你迈出第一步。

## 写在后面

讲一讲最近看到的一些内容：

关于 AI 到底在替代人还是在成就人，关于初级岗位会不会被彻底抹平，关于正在学计算机的人到底该往哪走。每个人都有自己的答案，但共识是：这个问题本身已经没法回避了。

---

1950 年，图灵问了一个问题：「机器能思考吗？」

他设计了一个测试。如果一台机器能在对话中让人类无法分辨它是人还是机器，那我们就应当承认它在思考。这就是图灵测试。

七十多年后，GPT-5.2 已经可以通过写代码搞定图灵测试了。Claude Code 能在一天之内完成你一周的工作。

你用这本指南走完整个项目，大概只需要一到两天。做完的那一刻，你大概率会被震撼到——这很正常。每个人都这样。

但图灵当年问的那个问题，其实还没被回答完。

机器的确表现得像在思考了。那反过来问：**当机器学会了思考，人应该做什么？**

当 AI 把执行力拉平之后，一百个人用同样的工具，产出可以差一百倍。差的不是 Prompt 谁写得好，不是 Skill 谁更花哨。

差的是那个人脑子里装了多少东西。他怎么理解这个世界，他的审美在哪一层，他能不能分辨什么是好的。这些事，AI 替不了你。

还有一件事值得想一想。

对于一个踩过十年坑、做过架构、带过团队的资深开发者来说，AI 是杠杆。那些他早就烂熟于心的重复劳动被接管之后，他的精力可以全部放在更高维度的决策上。AI 让他更强。

但对于一个刚入门的初学者呢？

那些被 AI 优雅跳过的"麻烦事"——手写表单校验、反复调 CSS 布局、被一个路径引用卡住半小时——在资深开发者眼里是低效劳动，但对初学者来说，恰恰是这些麻烦事在帮他建立对代码的感知。你亲手踩过一百个坑，才会在第一百零一个坑面前直觉般觉得"这里不太对"。这份直觉不是在书本上读来的，是在一遍一遍的试错中磨出来的。

**AI 代替大脑，Token 接管思考。** 

当 AI 替初学者接管了所有本该属于他的磨砺过程，他从哪里去接触真实的业务、从哪里去慢慢长出那种"说不清为什么但总觉得哪里不对"的意识？这个问题没有标准答案。但每一个正在入门的人，都值得问自己一遍。

计算机这行有一点好：它不太看你的出身，只看你能不能把东西做出来。AI 把这个门槛又往下压了一截，很多以前要学三年的技能，现在聊着天就做完了。这当然是好事。但门槛越低，门里面的人和人之间的差距就越大。

工具让所有人都站在了同一条起跑线上。但跑起来之后，谁快谁慢，就跟工具没关系了。

所以在这个混沌的时代里，找到你想成为什么样的人。然后往前走，别回头。

永远对世界保持好奇。

## 这是什么

一本以**搭建个人网站**为完整实践案例的 Vibe Coding 协作手册。你跟着做完一个网站的同时，会自然理解：

- 怎么跟 AI 说清楚你要什么
- 怎么把一个项目拆成 AI 能消化的步骤
- 怎么在 AI 写出 bug 时不崩溃
- 怎么判断自己现在在什么水平、下一步该练什么

**不教具体技术，教的是方法。** 换一个框架、换一个 AI 工具，这套方法依然有效。

## 适合谁

- 第一次用 AI 做项目，不知道从哪下手
- 用过 AI 但感觉"时好时坏"，说不上为什么
- 想把自己和 AI 的协作方式系统化

## 安装与使用

### 方法 1：安装为 Skill（推荐，可自动激活）

将下面这句发给你的 AI 编程 Agent（Claude Code、Cursor、Copilot、Codex 等均适用）：

```
帮我安装一个 skill，地址是 https://github.com/gulugulu123A/vibe-coding-guide-skills ，装好之后引导我从里程碑 0 开始。
```

Agent 会自动 clone 到 skills 目录。之后每次启动时，只要你的需求跟搭网站相关，Agent 会识别 `SKILL.md` 中的元数据并自动激活。

### 方法 2：终端手动安装

```bash
# 克隆到 Agent 的 skills 目录（以 Claude Code 为例）
git clone https://github.com/gulugulu123A/vibe-coding-guide-skills.git ~/.claude/skills/vibe-coding-guide-skills
```

重启你的 AI 编程工具后生效。也可以手动激活：

```
/skill vibe-coding-guide-skills
```

### 方法 3：不用 Skill，直接当文档用

```bash
git clone https://github.com/gulugulu123A/vibe-coding-guide-skills.git
cd vibe-coding-guide-skills
```

用你喜欢的 AI 工具打开这个目录，告诉它：

> 参考这个目录里的文档，引导我从里程碑 0 开始搭一个个人网站。

## 核心理念

> 技能可以被 AI 补上，但协作方法不行。Vibe Coding 的本质不是"让 AI 写代码"，而是"你知道怎么让 AI 写出你想要的东西"。

这本指南融合了三个底层框架：

1. **六个底层特质**（好奇心、靠谱、事实洁癖、多元化思维、忍受不确定性、低 ego 高自驱）——决定你在 AI 时代能不能走得远
2. **十个使用等级**（Lv.0 旁观者 → Lv.10 一人军团）——帮你看清自己在哪、下一步往哪走
3. **内容创作三步法**（获取信息 → 找角度 → 创作）——做网站和写文章一样，角度比执行更重要

## 推荐搭配的 Skill

以下是实际搭建过程中验证过的优质 Skill，按使用阶段推荐。每个都是独立的开源项目，**可选安装，非本项目依赖**。

### `frontend-design`
- **功能**：生成高质量、有辨识度的前端界面，避免通用 AI 审美
- **优点**：产出风格独特，不会千篇一律；内置多种设计语言参考
- **缺点**：生成代码量较大，需要筛选；简单项目可能用力过猛

### `ui-ux-pro-max`
- **功能**：50+ 设计风格、161 套配色、57 组字体搭配、99 条 UX 准则
- **优点**：相当于随身设计顾问，不用自己查配色和字体；覆盖 10 个技术栈
- **缺点**：规则较多，初次使用需要花时间了解；部分风格偏欧美审美

### `gsap-core` + `gsap-react`
- **功能**：专业级 JS 动画引擎，滚动驱动动画、弹性缓动、时间线编排
- **优点**：CSS 做不了的效果它能做（staggered reveal、3D tilt、spring 弹窗）；性能优秀
- **缺点**：有学习曲线；需要额外注册 ScrollTrigger 等插件；部分高级插件需付费

### `shadcn/ui`
- **功能**：源码级开源组件库，按钮/弹窗/表单/菜单等开箱即用
- **优点**：代码直接进项目而非 node_modules，完全掌控；社区活跃
- **缺点**：仅支持 React/Vue；初始配置需引入 Tailwind CSS

### `neat-freak`
- **功能**：会话结束后自动审查项目文档、同步 CLAUDE.md、清理过时记忆
- **优点**：防止文档腐烂，确保下次对话 AI 能正确接上；OCD 级的整理标准
- **缺点**：整理过程较耗时；可能需要多轮交互确认
