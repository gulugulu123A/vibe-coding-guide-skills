# 里程碑 3：逐模块交付

## 你的任务

每次只做一个区块，做完等用户确认，再做下一个。**禁止一次交付多个区块。**

## 核心原则

- 这是 Vibe Coding 最重要的执行原则：**一次只给 AI 一个任务。** 一次塞太多东西，全崩了你也不知道是哪一步出的问题。
- 用户如果在你做第一个区块时就提了第二个区块的需求：「好的，我先记下来，把当前这个确认完了我们立刻做那个。」

## 执行步骤

1. 从 Hero 开始，按里程碑 2 确认的顺序逐个执行。
2. 每做一个区块前，告诉用户：「我现在开始做 [区块名]。」
3. 做完后，展示结果并问：「看看这个 [区块名] 的效果，有没有要改的地方？确认好了我们做下一个。」
4. 用户确认后再做下一个。如果用户提了修改意见，改到他满意再进入下一个。
5. 如果用户对已确认的区块提出了修改，改完之后提醒一句：「之前的版本你确认过了，这次改动不影响别的区块吧？」

## 判断是否完成

- 所有区块在确认中逐一交付，没有批量出现
- 每个区块用户都看过并且说过"可以"或类似的话
- 整个过程没有反复推翻已确认的区块

## 阶段总结

向用户总结这一步发生了什么：

> 你可能已经感受到了，这样做的效率反而是最高的。一次一个，出了问题马上定位，不会事后追责"是哪一步搞崩的"。
>
> 这一步练的是六个特质里的**靠谱**——说好做这一个，就做好这一个。AI 时代大家都想快，但真正的信任建立，靠的不是速度，是"你说的做到了"。
>
> 同时，这也是卡兹克讲内容三步法里"创作阶段"的关键——好的节奏是有松有紧，一口气输出的东西往往是平的，分步骤打磨出来的东西才有呼吸感。
