# 参考：部署与域名方案

## 何时加载

在里程碑 0（需求诊断）或里程碑 1（首次部署）中，如果用户对部署和域名没有任何概念，加载本文件。

## 你的任务

用最通俗的语言帮用户理解三件事：什么是域名、什么是部署、他有哪些选择。**不要一次性全讲完，在他需要用到的那个节点讲对应的部分。**

## 背景知识（用类比讲给用户）

> 搭网站就像开店。部署就是租店面——你得有个地方把网页文件放上去让别人访问。域名就是店招牌——别人通过这个地址找到你。你可以先有店面再挂招牌，也可以两个一起搞定。

## 方案一：零成本方案（推荐新手）

**GitHub Pages + 免费二级域名**

- 完全免费，不需要买任何东西
- 你的网站地址是 `你的用户名.github.io`
- 适合个人作品集、博客、项目展示
- 操作：「你把代码推到 GitHub 仓库，GitHub Pages 自动帮你部署。不用管服务器、不用配环境，推上去就有了。」

> 这是最适合新手的起点。先跑通，如果以后觉得需要自己的域名，再买一个绑上来就行。前面的工作都不白费。

## 方案二：零成本 + 自定义域名

**Vercel / Netlify + 免费二级域名**

- 和 GitHub Pages 类似，但部署更灵活，支持更多框架
- Vercel 每天有 100 次免费部署次数限制
- 自动 HTTPS，全球 CDN 加速
- 「告诉 Agent 把代码推到 GitHub，Vercel 会自动检测并部署。部署完你就能拿到一个链接。」

## 方案三：买个自己的域名

**腾讯云 / 阿里云 / Cloudflare 购买域名**

- `.com` 域名大概 50-80 元/年，`.cn` 大概 30-50 元/年
- 买了域名之后，可以绑定到上面的 GitHub Pages 或 Vercel 上
- 「域名是你自己的，以后换服务器也不用换地址。个人网站用一个自己的域名，比分发 `xxx.github.io` 好看很多。」

如果用户想买域名，引导步骤：

1. 打开腾讯云或阿里云官网，搜索域名注册
2. 输入你想要的名字，看有没有被占用
3. 选一个没被占的，下单付款（需要实名认证）
4. 买完之后，按 GitHub Pages 或 Vercel 的教程把域名绑上去（你帮用户查具体步骤）
5. 等 DNS 生效（通常 10 分钟到几小时不等）

## 方案四：国内服务器方案

**腾讯云轻量应用服务器 / 阿里云 ECS**

- 适合需要后台、数据库、用户系统的网站
- 轻量服务器大概 50-100 元/月起步
- 需要备案（国内服务器强制要求，流程大概 2-4 周）
- 「如果你只是做个人展示网站，不建议走这条路。零成本方案完全够用。等你真的需要后台了再考虑。」

## 引导原则

- 用户说"不知道"时，默认推方案一（GitHub Pages），最省心
- 用户想有自己的域名时，推方案三但不催他。域名是几十块钱的事，什么时候买都行
- 用户提到"后台""数据库""用户系统"时，才介绍方案四
- 不要让用户觉得他在做一件很复杂的事。搭网站从来不是技术问题，是做决策的问题
