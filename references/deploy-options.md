# 参考：部署、域名与常见基础设施问题

## 何时加载

在里程碑 0（需求诊断）或里程碑 1（首次部署）中，如果用户对部署、域名、服务器等概念没有了解，或者在后续任何里程碑中用户遇到了基础设施相关的问题，加载本文件。

## 你的任务

用最通俗的语言帮用户理解他遇到的问题和可选方案。**不要一次性全讲完，在用户需要用到的那个节点讲对应的部分。**

## 最重要的原则

**当遇到你无法 100% 确定是最优方案的时候——尤其是涉及付费产品——必须提醒用户：**

> 我给出的方案是基于我所知道的信息。但云服务的价格、政策、活动经常变化，尤其是涉及付费的时候，建议你自己去 B 站、小红书或者浏览器上搜一下最新的对比评测，花十分钟查一下能避很多坑。

**这个提醒不是推卸责任，是对用户的钱负责。**

以下场景必须触发这个提醒：
- 推荐了具体的付费产品（域名、服务器、CDN 等）
- 提到了具体的价格数字
- 用户问"这个和那个哪个更好"
- 涉及备案、政策等可能随时变动的信息
- 用户要求对比多个云服务商

## 背景知识（用类比讲给用户）

> 搭网站就像开店。部署就是租店面——你得有个地方把网页文件放上去让别人访问。域名就是店招牌——别人通过这个地址找到你。服务器就是店面本身——它决定了你的店能承载多少客人。HTTPS 就是你的营业执照——证明你是正规的。备案是国内开店需要的行政审批。CDN 就是连锁分店——让全国各地的人都就近访问，不用全跑到总店来。

## 方案一：零成本方案（推荐新手）

**GitHub Pages + 免费二级域名**

- 完全免费，不需要买任何东西
- 你的网站地址是 `你的用户名.github.io`
- 适合个人作品集、博客、项目展示
- 自带 HTTPS，不需要额外配置

## 方案二：零成本 + 更灵活的部署

**Vercel / Netlify + 免费域名**

- 和 GitHub Pages 类似，但支持更多框架（Next.js、Nuxt 等）
- Vercel 免费额度：每月 100GB 带宽、每天 100 次部署
- 自动 HTTPS，全球 CDN

## 方案三：买个自己的域名

**腾讯云 / 阿里云 / Cloudflare 购买域名**

- `.com` 大概 50-80 元/年，`.cn` 大概 30-50 元/年（实际价格随时变化）
- 买了域名之后，可以绑定到 GitHub Pages 或 Vercel 上
- Cloudflare 域名注册不赚钱，按成本价卖，有时候最便宜

买域名的引导步骤：
1. 去腾讯云、阿里云或 Cloudflare 官网搜索想要的域名
2. 确认域名没被占用，下单付款（国内平台需要实名认证）
3. 到 GitHub Pages 或 Vercel 后台配置自定义域名
4. 在域名服务商那里添加对应的 DNS 解析记录
5. 等 DNS 全球生效（10 分钟到几小时，最长 48 小时）

## 方案四：国内服务器

**腾讯云轻量应用服务器 / 阿里云 ECS**

- 适合需要后台、数据库、用户系统的网站
- 轻量服务器 50-100 元/月起步（新用户通常有折扣）
- **强制备案**：国内服务器必须做 ICP 备案，流程 2-4 周，需要提交身份信息和网站信息
- 备案期间网站不能上线——提前规划好时间
- 境外服务器（如 AWS、Vercel、GitHub Pages）不需要备案

## 用户可能遇到的常见问题

### DNS 相关

「为什么我绑了域名打不开？」

DNS 解析需要时间生效，不是即时切换的。告诉用户：
> 刚改完 DNS 记录之后，全球生效通常 10 分钟到几小时，最长可能 48 小时。你可以过十分钟再试试，别着急。

### HTTPS / SSL 证书

「为什么浏览器提示网站不安全？」

网站没有配置 HTTPS 证书。GitHub Pages 和 Vercel 免费自带，不需要管。如果用户自己买了服务器，需要帮他配置 Let's Encrypt 免费证书。

### 备案

「我的网站为什么访问不了？」

如果用户用的是国内服务器且没有备案，这是唯一原因。告诉他：
> 国内服务器强制要求备案才能开放访问。你有两个选择：一是在服务器提供商后台提交备案申请，大概 2-4 周；二是暂时用 GitHub Pages 或 Vercel 这类不需要备案的境外平台。

### 域名实名认证

「为什么我买了域名不能用？」

国内购买的域名需要实名认证后才能正常解析。去购买平台提交身份证信息，通常 1-3 个工作日审核。

### 部署次数限制

「为什么今天部署不了了？」

Vercel 免费额度每天 100 次。提醒用户：
> 如果只是改文字内容，commit 时加上 `[skip ci]` 可以不触发部署。攒够了内容再统一部署一次。

### GitHub 访问不了

「我打不开 GitHub」

国内部分网络环境访问 GitHub 不稳定。建议：
> 你可以去 B 站或知乎搜一下最新的 GitHub 访问方案，关键词"GitHub 国内访问"。这个情况变化比较快，请以你自己查到的最近时间的信息为准。

### 云服务选哪家

「腾讯云和阿里云哪个好？」

> 对于个人网站来说，两家差距不大。你可以去 B 站或小红书上搜一下近期的轻量服务器对比视频，看哪个当前的活动更划算。新用户通常有首单折扣，这比任何技术对比都实际。

## 引导原则

- 用户说"不知道"时，默认推方案一（GitHub Pages），零成本零配置
- 用户想有自己的域名，推方案三但不催，域名什么时候买都行
- 用户提到"后台""数据库"时，才介绍方案四，同时必须提醒备案的时间成本
- 不要把"备案""DNS""HTTPS"堆在一起讲，碰到哪个讲哪个
- **涉及付费方案时，必须附加自查提醒**
