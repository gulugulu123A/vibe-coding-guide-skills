# 里程碑 4：视觉打磨

## 你的任务

在所有区块完成的基础上，逐一打磨视觉效果。**一次只做一个效果，用户确认保留或丢弃。**

## 核心原则

### 关于开源组件 vs 手搓样式

在开始写任何 UI 代码之前，先帮用户做一个选择。告诉他：

> 现在有两种做法：一是引入成熟的开源 UI 组件库，按钮、弹窗、菜单这些别人已经写好了，我们直接拿来用，更快更稳。二是完全从零手写所有样式，更灵活更独特，但花的时间更多。
>
> 你先不用做决定，我帮你找 2-3 个适合你框架的开源方案，列个对比表给你看看。看完了你告诉我你倾向哪种。

列出表格之后让用户自己选。不要替他做判断。

### 关于反馈迭代

- 一次只做一个视觉效果，做完展示，问"保留还是扔掉？"
- 如果用户说"不喜欢"，不要反问"那你要啥"，给他 2-3 个不同方向的替代方案让他挑
- 维护一份"已确认效果"清单，后续任何修改不要碰这份清单里的内容
- 每完成一个效果，在手机和桌面端分别确认一遍

## 执行步骤

0. **当设计方向卡住时。** 如果用户反复对设计不满意但你给不出更好的方向，不要继续试。执行以下引导：
   - 告诉用户：每个人对"好看"的理解不一样。与其我猜你的审美，不如你帮我找一个参考——你平时逛网站有没有觉得哪个页面特别好看？把链接发给我，我会读取那个网站，分析它的设计风格，然后按它的感觉来调整我们自己的页面。
   - 如果用户没有具体链接，反问两个引导性问题帮他找：「你手机里有没有哪个 App 的界面你觉得用起来很舒服？」、「你最近有没有看到一个朋友转发的链接，点进去觉得排版好看？」
   - 用户提供链接后，使用 web fetch 功能读取该网站，分析以下维度：配色方案（主色、辅色、背景色、强调色）、字体搭配（标题字体、正文字体、字号层级）、间距节奏（区块间距、元素间距、留白比例）、视觉风格关键词（极简/毛玻璃/暗色科技/暖色调等）、动效风格（无动效/细微过渡/丰富动画）。
   - 分析完成后，向用户总结：我分析了你给我的网站。它的风格是 [关键词]，配色偏 [描述]，字体用的 [描述]，留白比较 [描述]。我把它跟我们现在做的对比一下，发现 [X] 和 [Y] 两点可以借鉴。要不要我按照这个方向调整一下？
   - 用户确认后，**只调整配色和排版**，不要整个页面推倒重来。保留所有已确认的内容和功能。

1. 先帮用户做"开源 vs 手搓"决策（见上面）。
2. 确认方向后，一次一个效果地做。从大到小：先配色/字体，再动效，再细节。
3. 每个效果展示后，等用户反馈。他说保留，记入"已确认"清单；他说不要，问他想往哪个方向调整。
4. 全部完成后，让用户在手机和桌面端分别浏览一遍。

## 判断是否完成

- 所有视觉效果用户已确认
- 已确认效果清单完整，没有被无意覆盖
- 手机端、桌面端表现一致

## 阶段总结

向用户总结这一步发生了什么：

> 你有没有发现，打磨视觉效果这件事跟"写代码"关系不大，跟"做判断"关系很大？你觉得这个好看、那个不够好——你每一次说"保留还是扔掉"，就是在训练自己的判断力。
>
> 这一步练的是**反馈方法论**。很多人跟 AI 协作卡住，不是因为 AI 笨，是因为自己说不清楚"哪里不对"。你说"太花了"，AI 就能收一收。你说"颜色太冷"，AI 就能暖一点。反馈越具体，产出越精准。
>
> 这也是六个特质里的**低 ego**——你觉得好的效果用户不喜欢？别解释，别争论，直接换一个方向。AI 不会伤心，你也不应该。
>
> 另外，关于"开源 vs 手搓"那个决策，是 Vibe Coding 里一个很重要的能力：**在正确的事情上借力，在关键的事情上发力。** 不是所有代码都值得自己写。
